# Integrations

## Format Backends
`MZReaderType` aggregates all supported format backends behind a single API; each backend lives in a dedicated module (`src/io/mgf`, `src/io/mzml`, `src/io/mzmlb`, `src/io/imzml`, `src/io/thermo`, `src/io/tdf`) and is gated by the corresponding feature flag (`mgf`, `mzml`, `mzmlb`, `imzml`, `thermo`, `bruker_tdf`). mzML readers lean on `quick-xml`/`mzpeaks`, mzMLb needs `hdf5` + `ndarray`, Thermo uses the native `thermorawfilereader`, and Bruker TDF depends on `timsrust` + `rusqlite` for the SQL index file plus the specialized `TDFFrameReader`. The `MZReader` builder (in `src/io/infer_format/dispatch.rs`) routes a stream to the correct backend via `infer_format`, and the `IMMZReaderType` controls ion mobility frame integration when those feature flags are enabled.

## Cross-format EIC Integration
The shared `ExtractedIonChromatogramSource` trait in `src/io/eic.rs` is implemented by every reader and re-exported in `src/io/mod.rs`, so callers just import `mzdata::io::{ExtractedIonChromatogramSource, EICQuery}` and drive extraction through `MZReader::extract_eic(s)` no matter which backend is active. `src/io/infer_format/dispatch.rs` dispatches to the format-specific implementations and falls back to `extract_eics_from_spectra` when the reader is unknown or purely in-memory. Bruker TDF introduces a `TDFSpectrumReader::extract_eics_fast` path that works against the cached `entry_index` and `frame_reader` metadata for lower latency, but the API surface is identical from the caller's point of view. Additional `test` helpers (e.g., `manual_extract` in `src/io/infer_format/mod.rs`) reuse the same readers to validate correctness per backend.

## Automation & Testing Hooks
The `Justfile` ties into the integrations: `test-units`/`test-units-more` run `cargo nextest` with the broadest feature set (`thermo`, `mzmlb`, `async`, `numpress`, `bruker_tdf`, `parallelism`, `mzsignal`), while `test-coverage` drives `cargo llvm-cov` across the same stack. Doc generation (`quick-docs`, `docs`) and binary installs (`install-mzdata`) also share those feature combinations to keep docs/test coverage aligned with shipping binaries. The EIC change is exercised in `src/io/infer_format/mod.rs` tests (`test_extract_eic_dispatch_mzml`, `test_extract_eics_dispatch_batch_mzml`, `test_extract_eic_dispatch_tdf`), and the sample files under `test/data` (e.g., `small.mzML`, `diaPASEF.d`) serve as regression fixtures that the same automation commands build into every run.
