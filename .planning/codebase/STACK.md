# Stack

## Languages & Build Tools
The workspace is a Rust crate (`edition = "2021"`) built entirely with `cargo` and coordinated through the `Justfile`. `cargo nextest` is the default test runner (see `test-units`/`test-units-more`), `cargo llvm-cov` (via `test-coverage`) produces coverage artifacts, and `cargo doc`/`cargo install` steps reuse the same feature flags as the library. Release automation leans on `cliff`/`git` wrappers in `Justfile` scripts, while docs and `mzdata-spectra` rely on consistent version bump scripts in the same file.

## Feature Flags & Dependencies
The crate's facade lives under `Cargo.toml` with default features `zlib-ng-compat`, `mgf`, and `mzml`, plus opt-ins for `imzml`, `mzmlb`, `thermo`, `bruker_tdf`, `mzsignal`/`nalgebra` (and alternative math backends like `openblas`, `netlib`, `intel-mkl`), `parallelism` (Rayon), `checksum`, `async`, `proxi`, `zstd`, and more. Key dependencies include `mzpeaks` (spectrum math), `mzsignal` (signal processing), `quick-xml` (mzML parsing), `timsrust`/`rusqlite` (Bruker TDF), `thermorawfilereader` (Thermo RAW), `hdf5`/`ndarray` (mzMLb), and compression helpers in `flate2`. Optional async support leans on `tokio`/`async_partial`, while `serde`/`serde_json`/`serde_with` are gated for serialization.

## Cross-format Fast EIC
The new Extracted Ion Chromatogram (EIC) API lives in `src/io/eic.rs`, defining `EICQuery`, `ExtractedIonChromatogram`, and the `ExtractedIonChromatogramSource` trait. Each format reader (MGF, mzML, mzMLb, Thermo RAW, IMzML, Bruker TDF, memory sources) implements that trait and registers itself in the multi-format `MZReaderType` dispatch (`src/io/infer_format/dispatch.rs`), so `MZReader::extract_eic(s)` exposes a single entry point regardless of backend. `DetailLevel` is temporarily lowered (`Lazy`) while EICs iterate spectra, raw arrays, or cached frame data, and the Bruker `TDFSpectrumReader` adds a specialized `extract_eics_fast` loop over `entry_index`/`frame_reader` for the `bruker_tdf` feature. This layered trait + dispatch design keeps the fast-path TDF logic next to general array-based extraction and ties into the shared reader ecosystem the rest of the stack depends on.
